# AI Agent Working Agreement

This repository is developed from normative specifications and small executable tasks.

## Required reading order

Before changing code, read:

1. `docs/README.md`
2. `docs/09-execution/capability-matrix.md`
3. The assigned task file
4. Every specification and ADR listed by that task
5. The current code and tests in the task's declared scope

Before changing frontend layout, navigation, styling, transcript interaction,
chat presentation, or evidence presentation, also read
`docs/00-product/interface-specification.md`. Current frontend styling is not the
target visual authority.

## Sources of truth

When sources disagree, use this precedence:

1. Accepted ADRs for architectural decisions
2. Normative specifications for target behavior
3. OpenAPI generated by the running FastAPI application for the current HTTP schema
4. Tests and code for currently implemented behavior
5. Explanatory documentation

Target specifications do not prove that a capability is implemented. Only the capability matrix and passing acceptance tests may mark it `verified`.

## Execution rules

- Implement only one task at a time and satisfy its dependencies first.
- Do not invent missing product or architecture decisions. Record a blocker or propose an ADR.
- Preserve timestamp provenance throughout transcript, chunk, retrieval, and citation flows.
- Keep provider-specific SDK objects inside adapters.
- Never make a video `ready` when required ingest/index stages are incomplete.
- Do not silently fall back to a paid provider.
- Do not present decorative controls, fabricated progress, health, storage, or
  navigation states as working UI.
- Update tests, affected docs, and the capability matrix in the same commit.
- Run the task verification commands before committing directly to `main`.
- Keep commits small, atomic, and reversible. Do not modify unrelated user changes.

## Language

Technical documentation, code identifiers, comments, logs, and error codes use English. The product supports Vietnamese and English input/output according to `docs/03-specifications/multilingual-spec.md`.
